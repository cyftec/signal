Best Practices

Recommended practices for writing clean and maintainable signal code.

Always Access Signals in Effects

Effects only track dependencies for signals accessed during execution. Always access signals, even if you don't use the value:

// WRONG - count won't be tracked if shouldShow is false initially
const shouldShow = signal(false);
const count = signal(5);
effect(() => {
  if (shouldShow.value) {
    console.log(count.value);
  }
});

// CORRECT - Always access the signal
effect(() => {
  if (shouldShow.value) {
    console.log(count.value);
  } else {
    count.value; // Access to track dependency
  }
});

Use Ternary for Conditional Dependencies

For derived signals, use ternary operators to ensure dependencies are always accessed:

// WRONG
const displayed = derive(() => {
  if (shouldShow.value) {
    return count.value;
  }
  return 0;
});

// CORRECT
const displayed = derive(() => {
  return shouldShow.value ? count.value : 0;
});

Dispose When Done

Always dispose effects and derived signals when they're no longer needed to prevent memory leaks:

const count = signal(0);
const doubled = derive(() => count.value * 2);
const eff = effect(() => console.log(count.value));

// Dispose when cleanup is needed
dispose(doubled, eff);

Use Shallow Merge for Objects

Object signals use shallow merge with set(). For deep updates, replace the entire object:

const user = signal({ 
  name: "John", 
  address: { city: "NYC" } 
});

// Shallow merge - only updates top-level properties
user.set({ address: { city: "LA" } });
// user.value is { name: "John", address: { city: "LA" } }

// For deep updates, replace entire object
user.value = { 
  ...user.value, 
  address: { ...user.value.address, city: "LA" } 
};

Leverage Traps for Type Safety

Use traps to get type-specific methods that maintain type safety:

const count = signal(5);
const trapped = trap(count);

// Type-safe number operations
const doubled = trapped.mul(2).result;
const clamped = trapped.toConfined(0, 10).result;

Check Error First in promstates

Always check the error signal before using the result signal:

const [runPromise, result, error] = promstates(fetchData, null);

effect(() => {
  if (error.value) {
    console.error("Error:", error.value);
  } else {
    console.log("Result:", result.value);
  }
});

Avoid Type Changes After Creation

Trap and operation types are determined at creation time. Type changes in signals won't be reflected:

// AVOID - Type changes won't be reflected
const value = signal(5);
const trapped = trap(value); // NumberSignalTrap
value.value = "hello"; // Type change not reflected

// PREFERRED - Create new trap if type changes
const value = signal(5);
let trapped = trap(value);
// When type might change, recreate trap
if (typeof value.value !== "number") {
  trapped = trap(value);
}

Use Compute for Function Arguments

Use compute() when you need to pass signals as function arguments:

const a = signal(5);
const b = signal(3);

// Clean function-based derived signal
const sum = compute((x: number, y: number) => x + y, a, b);

Keep Effects Simple

Effects should focus on side effects. Keep computation in derived signals:

// PREFERRED - Computation in derived signal
const doubled = derive(() => count.value * 2);
effect(() => {
  updateUI(doubled.value);
});

// AVOID - Computation in effect
effect(() => {
  updateUI(count.value * 2);
});
