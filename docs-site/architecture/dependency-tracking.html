Dependency Tracking

Signal's global variable-based dependency tracking mechanism.

Global Variable-Based Tracking

Signal uses a module-level _currentSignalEffect variable instead of an explicit dependency graph.

How It Works

  1. When effect() is called, it sets _currentSignalEffect before executing the function
  2. When a signal's .value getter is called, it checks if _currentSignalEffect exists
  3. If it exists, the effect is added to the signal's internal _effects Set
  4. After effect execution, _currentSignalEffect is set back to null

Implementation

// Module-level variable
let _currentSignalEffect: SignalsEffect | null = null;

// Effect function
function effect(fn: () => void): SignalsEffect {
  const eff = fn as SignalsEffect;
  eff.canDisposeNow = false;
  
  _currentSignalEffect = eff;
  fn();
  _currentSignalEffect = null;
  
  return eff;
}

Signal Value Getter

The signal value getter checks for the current effect and registers dependencies:

// Simplified value getter
get value(): T {
  if (_currentSignalEffect) {
    this._effects.add(_currentSignalEffect);
  }
  return newVal(this._value);
}

No Explicit Graph Structure

Signal does not maintain an explicit dependency graph data structure.

  • No dependency graph data structure
  • Each signal maintains a Set of registered effects
  • Dependencies are tracked implicitly through the global variable during execution

Implications

  • Effects must access .value during execution to establish dependencies
  • Simpler implementation but less flexible for nested scenarios
  • Cannot track dependencies that aren't accessed during execution

Dependency Establishment

An effect or derived signal establishes a dependency on a signal only if the signal's .value getter is accessed during execution.

Important

If a signal's .value is accessed conditionally and the condition is false during initial execution, that signal will not be tracked as a dependency.

Example

// WRONG - count won't be tracked if shouldShow is false initially
const shouldShow = signal(false);
const count = signal(5);
const displayed = derive(() => {
  if (shouldShow.value) {
    return count.value;
  }
  return 0;
});

// CORRECT - Always access the signal
const displayed = derive(() => {
  return shouldShow.value ? count.value : 0;
});

No Automatic Dependency Collection

Signal does not automatically collect dependencies beyond what is accessed during execution.

Guarantees

  • Dependencies are established during the initial execution of an effect or derived signal computation
  • Once a dependency is established, the effect or derived signal will re-run whenever that signal's value changes
  • Dependencies are not automatically re-evaluated on subsequent executions

Non-Guarantees

  • No automatic dependency collection beyond what is accessed during execution
  • No guarantee that conditional dependencies will be tracked if the condition is false during initial execution
  • No guarantee about the internal mechanism of dependency tracking

Derived Signal Dependency Tracking

Derived signals use the same mechanism as effects for dependency tracking.

// Derived signals also use global variable tracking
function derive<T>(valueGetterFn: (oldValue: T | undefined) => T): DerivedSignal<T> {
  let oldValue: T | undefined = undefined;
  const derivedSource = signal(valueGetterFn(oldValue));
  
  // This effect uses the same tracking mechanism
  const derivedSourceUpdator = effect(() => {
    const newValue = valueGetterFn(oldValue);
    oldValue = newValue;
    derivedSource.value = newValue;
  });
  
  return {
    type: "derived-signal",
    oldValue,
    derivedSource,
    derivedSourceUpdator,
    get value() { return derivedSource.value; },
    get prevValue() { return oldValue; },
    dispose() { derivedSourceUpdator.dispose(); }
  };
}

Comparison with Other Libraries

Most signal libraries use explicit dependency graphs or stacks:

  • SolidJS - Uses explicit dependency graph with cleanup
  • Preact Signals - Uses explicit dependency tracking
  • Angular Signals - Uses explicit dependency graph
  • Signal - Uses global variable-based tracking

Trade-offs

  • Pros: Simpler implementation, less memory overhead
  • Cons: Less flexible for nested scenarios, requires explicit access pattern
