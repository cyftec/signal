Derived Values

Computed values that update automatically when their dependencies change.

Creating Derived Signals

Use the derive() function to create read-only derived signals from other signals.

derive<T>(valueGetterFn: (oldValue: T | undefined) => T): DerivedSignal<T>

Basic Example

const count = signal(5);
const doubled = derive(() => count.value * 2);

console.log(doubled.value); // 10
count.value = 10;
console.log(doubled.value); // 20

Value Getter Function

The value getter function receives the previous computed value as an argument.

const count = signal(5);
const history = derive((prev) => {
  const current = count.value;
  return prev ? [...prev, current] : [current];
});

console.log(history.value); // [5]
count.value = 10;
console.log(history.value); // [5, 10]

Previous Value

The prevValue getter provides access to the previous computed value.

const count = signal(5);
const doubled = derive(() => count.value * 2);

console.log(doubled.value); // 10
console.log(doubled.prevValue); // undefined

count.value = 10;
console.log(doubled.value); // 20
console.log(doubled.prevValue); // 10

Dependency Tracking

Derived signals track dependencies based on which signals are accessed during computation.

Important

Dependencies are only tracked if signals are accessed during the initial computation. Conditional access may not establish dependencies.

Correct Pattern

// CORRECT - Always access signals
const displayed = derive(() => {
  return shouldShow.value ? count.value : 0;
});

Chaining Derived Signals

Derived signals can depend on other derived signals, creating computation chains.

const count = signal(5);
const doubled = derive(() => count.value * 2);
const quadrupled = derive(() => doubled.value * 2);

console.log(quadrupled.value); // 20
count.value = 10;
console.log(quadrupled.value); // 40

Multiple Dependencies

Derived signals can depend on multiple source signals.

const firstName = signal("John");
const lastName = signal("Doe");
const fullName = derive(() => `${firstName.value} ${lastName.value}`);

console.log(fullName.value); // "John Doe"
firstName.value = "Jane";
console.log(fullName.value); // "Jane Doe"

Disposal

Call dispose() on a derived signal to stop it from tracking dependencies.

const count = signal(5);
const doubled = derive(() => count.value * 2);

doubled.dispose(); // Stops tracking dependencies
count.value = 10;
console.log(doubled.value); // Still 10 (won't update)

After disposal, the value remains accessible but won't update when dependencies change.

Edge Cases

No Signal Access

If the value getter function doesn't access any signal values, it runs only once.

const constant = derive(() => 42);
console.log(constant.value); // 42 (never changes)

Error Propagation

If the value getter function throws an error, the error propagates immediately.

const count = signal(5);
const risky = derive(() => {
  if (count.value < 0) throw new Error("Negative value");
  return count.value * 2;
});

Implementation Details

Derived signals are implemented as a source signal + effect pattern:

  • Internal source signal holds the computed value
  • Internal effect runs the value getter function
  • The effect updates the internal source signal with the computed value
  • When dependencies change, the effect re-runs and updates the value
