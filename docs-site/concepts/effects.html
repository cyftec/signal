Effects

Side effects that run when signals change.

Creating Effects

Use the effect() function to register a callback that runs when accessed signals change.

effect(fn: () => void): SignalsEffect

Basic Example

const count = signal(0);

effect(() => {
  console.log("Count is:", count.value);
});

count.value = 1; // Logs: "Count is: 1"
count.value = 2; // Logs: "Count is: 2"

Immediate Execution

Effects run immediately when created, not just when dependencies change.

const count = signal(0);

effect(() => {
  console.log("Count is:", count.value);
});
// Logs: "Count is: 0" immediately

Dependency Tracking

Effects track dependencies based on which signals are accessed during execution.

Important

Effects only register to signals if .value is accessed during execution. Conditional access may not establish dependencies.

Correct Pattern

// CORRECT - Always access signals
const shouldLog = signal(false);
const count = signal(0);

effect(() => {
  if (shouldLog.value) {
    console.log(count.value);
  } else {
    count.value; // Access to track dependency
  }
});

Multiple Dependencies

Effects can track multiple signals by accessing their values.

const firstName = signal("John");
const lastName = signal("Doe");

effect(() => {
  console.log(`${firstName.value} ${lastName.value}`);
});

firstName.value = "Jane"; // Effect runs
lastName.value = "Smith"; // Effect runs

Synchronous Execution

Effects run synchronously when their dependencies change.

const count = signal(0);
let runCount = 0;

effect(() => {
  runCount++;
  count.value;
});

console.log(runCount); // 1 (immediate execution)
count.value = 1;
console.log(runCount); // 2 (synchronous update)
count.value = 2;
console.log(runCount); // 3 (synchronous update)

Disposal

Call dispose() on an effect to mark it for cleanup.

const count = signal(0);
const eff = effect(() => {
  console.log(count.value);
});

eff.dispose(); // Marks for disposal
count.value = 1; // Effect is removed without running

Lazy Cleanup

Disposed effects are removed on the next signal update, not immediately.

const count = signal(0);
const eff = effect(() => console.log(count.value));

eff.dispose();
// Effect is still registered until next signal update
count.value = 1; // Effect is removed here

Nested Effects

Effects can be nested within other effects.

const count = signal(0);

effect(() => {
  console.log("Outer:", count.value);
  
  effect(() => {
    console.log("Inner:", count.value);
  });
});

Error Handling

If an effect throws an error, the error propagates immediately and may prevent subsequent runs.

const count = signal(0);

effect(() => {
  try {
    if (count.value < 0) throw new Error("Negative value");
    console.log(count.value);
  } catch (error) {
    console.error("Effect error:", error);
  }
});

Edge Cases

No Signal Access

If an effect doesn't access any signal values, it runs only once.

effect(() => {
  console.log("Runs once");
});
// Never runs again

Same Value Updates

Effects don't run if a signal is set to the same value.

const count = signal(0);
let runCount = 0;

effect(() => {
  runCount++;
  count.value;
});

count.value = 0; // No change, effect doesn't run
console.log(runCount); // 1
