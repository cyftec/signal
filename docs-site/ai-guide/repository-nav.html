Repository Navigation Guide

Navigate the Signal repository structure effectively.

Repository Structure

signal/
├── src/
│   ├── _core/           # Core primitives
│   │   ├── signal/      # Source signal implementation
│   │   ├── derive.ts    # Derived signal implementation
│   │   ├── effect.ts    # Effect implementation
│   │   ├── dispose.ts   # Disposal utility
│   │   └── utils/       # Type checkers and utilities
│   └── api/             # Higher-level APIs
│       ├── operations/  # Composable operations
│       ├── traps/       # Type-specific utilities
│       ├── connectors.ts # Signal connections
│       ├── template.ts  # String interpolation
│       ├── promstates.ts # Promise states
│       └── compute.ts   # Function-based derived signals
├── tests/               # Test suite
├── docs/                # Documentation
│   ├── architecture.md  # Architecture notes
│   ├── behavior.md      # Behavioral inventory
│   ├── semantics.md     # Semantics and guarantees
│   └── llm-guide.md     # AI guide (this file)
└── package.json

Key Files

Core Implementation

  • src/_core/signal/ - Source signal implementation with array/object extensions
  • src/_core/derive.ts - Derived signal implementation
  • src/_core/effect.ts - Effect implementation with global tracking
  • src/_core/dispose.ts - Disposal utility function

API Layer

  • src/api/operations/ - Composable operations (op)
  • src/api/traps/ - Type-specific utilities (trap)
  • src/api/connectors.ts - Signal-to-signal connections (transmit, receive)
  • src/api/template.ts - String interpolation (tmpl)
  • src/api/promstates.ts - Promise state management
  • src/api/compute.ts - Function-based derived signals

Documentation

  • docs/architecture.md - High-level architecture and design decisions
  • docs/behavior.md - Complete API behavioral inventory
  • docs/semantics.md - Guarantees and non-guarantees
  • docs/llm-guide.md - Comprehensive AI guide

Tests

  • tests/signal.test.ts - Core signal tests
  • tests/derive.test.ts - Derived signal tests
  • tests/effect.test.ts - Effect tests
  • tests/operations.test.ts - Operations tests
  • tests/traps.test.ts - Trap tests
  • tests/connectors.test.ts - Connector tests
  • tests/promstates.test.ts - Promise states tests

Navigation Strategies

Understanding an API

  1. Start with docs/behavior.md for behavioral documentation
  2. Check corresponding test file for examples
  3. Review implementation in src/_core/ or src/api/
  4. Refer to docs/semantics.md for guarantees

Debugging Issues

  1. Check docs/semantics.md for non-guarantees
  2. Review docs/architecture.md for implementation details
  3. Check docs/llm-guide.md common mistakes section
  4. Examine relevant test cases for expected behavior

Code Generation

  1. Use docs/llm-guide.md as primary reference
  2. Consult docs/behavior.md for API details
  3. Check docs/semantics.md for behavioral guarantees
  4. Review docs/architecture.md for implementation constraints
