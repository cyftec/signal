High-Level Overview

Signal's major concepts and architectural decisions.

Three Main Primitives

Signal provides three main primitive types for reactive state management:

  • Source Signals - Mutable signals created from plain values
  • Derived Signals - Read-only signals computed from other signals
  • Effects - Functions that run when dependent signals change

Global Variable Dependency Tracking

Signal uses a global variable-based dependency tracking system rather than an explicit graph structure. This is a key architectural difference from many other signal libraries.

How It Works

  1. When effect() is called, it sets a module-level _currentSignalEffect variable before executing the function
  2. When a signal's .value getter is called, it checks if _currentSignalEffect exists
  3. If it exists, the effect is added to the signal's internal _effects Set
  4. After effect execution, _currentSignalEffect is set back to null

Implications

  • Simpler implementation but less flexible for nested scenarios
  • No explicit dependency graph - each signal maintains a Set of registered effects
  • Dependencies are tracked implicitly through the global variable during execution

No Explicit Graph Structure

Signal does not maintain an explicit dependency graph data structure.

  • Each signal maintains a Set of registered effects
  • Dependencies are tracked implicitly through the global variable during execution
  • This means effects must access .value during execution to establish dependencies

Synchronous Execution

Signal has no batching or deferred execution. All updates are synchronous and immediate.

  • When a signal's value is set, all registered effects run immediately
  • Effects run in the order they were registered
  • No scheduling queue or microtask timing

Note

This is different from SolidJS, Preact Signals, and Angular Signals, which all have batching mechanisms.

Derived Signal Implementation

Derived signals are implemented as a source signal + effect pattern, not as dedicated derived signal nodes.

  • Internal source signal holds the computed value
  • Internal effect runs the value getter function
  • The effect updates the internal source signal with the computed value
  • More indirect but achieves the same result

Array Mutation Methods

Signal provides array mutation methods that create new arrays internally but feel mutable to the user.

  • Includes standard methods: push, pop, shift, unshift, splice, reverse, sort, fill, copyWithin
  • Includes custom remove() method (inverse of filter)
  • Methods create new immutable arrays internally via @cyftech/immutjs
  • Different from most signal libraries that treat arrays as immutable

Type Discrimination

Signal uses a type property on all signal objects for runtime type checking.

  • Source signals have type: "source-signal"
  • Derived signals have type: "derived-signal"
  • Non-signal objects have type: "non-signal"
  • Enables runtime type checking for MaybeSignalValue types
  • Not common in other signal libraries

Non-Signal Objects

Signal provides explicit runtime type wrapper for plain values.

  • Created via getNonSignalObject(input: T)
  • Used for runtime type discrimination in complex type scenarios
  • Uncommon pattern in other libraries

API Layer Structure

The library has two main layers:

Core Layer (src/_core/)

  • Primitive implementations: signal, derive, effect, dispose
  • Type definitions and utilities
  • Type checkers and value getters

API Layer (src/api/)

  • Higher-level convenience APIs
  • Operations: composable operations on signals
  • Traps: type-specific utility methods
  • Connectors: signal-to-signal connections
  • Templates: string interpolation with signals
  • Promise states: async state management
  • Compute: function-based derived signals

Dependencies

Signal has minimal external dependencies:

  • @cyftech/immutjs - Used for immutable value handling
    • immut() - Creates immutable copy of value
    • newVal() - Extracts value from immutable wrapper
    • isPlainObject() - Type checking for plain objects
