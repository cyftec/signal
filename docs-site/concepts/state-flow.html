State Flow

How state changes propagate through the Signal system.

Data Flow Overview

Signal follows a unidirectional data flow pattern where changes propagate from source signals through derived signals to effects.

Flow Stages

  1. Signal Creation - Create source signals with initial values
  2. Effect Registration - Effects register dependencies by accessing signal values
  3. Value Update - Setting a signal value triggers registered effects
  4. Derived Signal Computation - Derived signals recompute when dependencies change
  5. Effect Execution - Effects run synchronously when their dependencies change

Signal Creation

Source signals are created with immutable initial values using the signal() function.

const count = signal(0);
const name = signal("John");

Array and object signals get extended with mutation methods:

const items = signal([1, 2, 3]);
const user = signal({ name: "John", age: 30 });

Effect Registration

Effects register dependencies by accessing signal values during their initial execution.

const count = signal(0);

effect(() => {
  // Accessing count.value registers this effect as a dependency
  console.log(count.value);
});

The effect is added to the signal's internal _effects Set when .value is accessed.

Value Update

Setting a signal value triggers all registered effects synchronously.

count.value = 1; // Triggers all registered effects

Update Process

  1. Signal checks if new value equals old value (short-circuits if equal)
  2. Signal updates internal value immutably using @cyftech/immutjs
  3. Signal iterates through its _effects Set
  4. For each effect: if canDisposeNow is true, remove it; otherwise, execute it
  5. Effects may access other signals' .value, triggering their effects recursively

Derived Signal Computation

Derived signals recompute their values whenever tracked dependencies change.

const count = signal(5);
const doubled = derive(() => count.value * 2);

count.value = 10; // doubled recomputes to 20

Implementation

Derived signals are implemented as a source signal + effect pattern:

  • Internal source signal holds the computed value
  • Internal effect recomputes the value when dependencies change
  • The effect runs the value getter function and updates the internal source signal

Effect Execution

Effects run synchronously when their tracked dependencies change.

const count = signal(0);

effect(() => {
  console.log("Count:", count.value);
});

count.value = 1; // Effect runs immediately
count.value = 2; // Effect runs immediately again

Execution Guarantees

  • Effects run immediately when created via effect()
  • Effects run synchronously when tracked signals change
  • Effects run exactly once per signal value change (assuming the value actually changes)
  • Effects do not run if the signal's value is set to the same value

Recursive Triggering

Effects can trigger other effects recursively by accessing other signals' values during execution.

const count = signal(0);
const doubled = derive(() => count.value * 2);

effect(() => {
  console.log("Count:", count.value);
  console.log("Doubled:", doubled.value);
});

count.value = 1;
// First effect runs, accessing doubled.value
// This triggers doubled's internal effect to recompute
// All happens synchronously

Disposal Cleanup

Disposed effects are removed lazily on the next signal update.

const count = signal(0);
const eff = effect(() => console.log(count.value));

eff.dispose(); // Marks for disposal
count.value = 1; // Effect is removed here without running
